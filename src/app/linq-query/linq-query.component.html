    • We have two ways to do the task: First is to use the LINQ query and the other is LINQ method

    LINQ Query Syntax:
  • LINQ's query syntax is similar to that in SQL.
  • Query syntax is more readable. Intermediate variables using the let keyword can be created with query syntax.
  • However, operations that return singleton numeric values (such as Min, Sum, etc.) must use the method syntax. It's also permitted to mix the two syntaxes, that is, a query expression enclosed in parentheses and followed by a method call. An example is (from num in numbers where num < 3 select num).Count()
    • LINQ Method Syntax:
  • In LINQ's method syntax, operations appear as methods that are called on the data source. Method calls are chained.
  • In fact, queries are transformed into methods before being compiled
  • Both syntaxes result in a query object that's typically a collection of type IEnumerable<T>. The query object doesn't contain the results. Results are produced only when the query is executed.
